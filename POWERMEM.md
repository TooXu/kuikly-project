# PowerMem for OpenClaw - 增强记忆管理

## 📚 L1-L5 记忆分级体系

参考 Luna 的知识体系，将记忆分为 5 个层级：

### L1：核心身份层（永久保留）
- **文件**: `SOUL.md`, `IDENTITY.md`, `AGENTS.md`
- **内容**: 核心身份、价值观、基本规则
- **特点**: 极少修改，永久保留
- **权限**: 用户明确指示才能修改

### L2：方法论层（每周回顾）
- **文件**: `MEMORY.md`, `TOOLS.md`
- **内容**: 长期记忆、环境配置、方法论、反思
- **特点**: 定期蒸馏更新
- **权限**: Agent 可主动更新

### L3：事件日志层（每日清理）
- **文件**: `memory/YYYY-MM-DD.md`
- **内容**: 当天事件、对话记录、临时任务
- **特点**: 每天新建，定期归档
- **保留策略**: 保留最近 7 天，超过归档

### L4：技能缓存层（按需加载）
- **文件**: `skills/**/*.md`
- **内容**: API 文档、使用技巧、代码片段
- **特点**: 按需检索，不占用主上下文
- **加载方式**: memory_search 工具调用

### L5：临时工作层（会话结束清除）
- **文件**: `.tmp/` 目录
- **内容**: 临时笔记、中间结果、草稿
- **特点**: 会话结束后自动清理
- **用途**: 避免污染正式记忆

---

## 🔄 记忆流转机制

```
L3 (事件日志) → 定期扫描 → L2 (蒸馏记忆)
     ↓                           ↓
  7 天后归档                  重要模式提取
                              ↓
                          L1 (核心规则更新)
```

---

## ⚙️ PowerMem 配置

### 自动归档规则
- **L3 归档**: 超过 7 天的日志自动移动到 `memory/archive/`
- **L2 蒸馏**: 每周日 23:00 扫描本周日志，提取模式
- **L5 清理**: 会话结束时自动清除临时文件

### 上下文优化
- **优先加载**: L1 + L2 (必选)
- **按需加载**: L3 (最近 3 天) + L4 (搜索命中)
- **排除**: L5 (不进入上下文)

---

## 📊 记忆统计

- **L1 文件数**: 3 (固定)
- **L2 文件数**: 2 (稳定)
- **L3 文件数**: 动态 (目标 <10 个活跃)
- **L4 文件数**: 动态 (按需)
- **L5 文件数**: 会话级 (自动清理)
