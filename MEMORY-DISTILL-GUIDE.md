#  OpenClaw 记忆蒸馏指南

## 🎯 目标

将日常对话日志中的**临时信息**提炼为**长期记忆**，减少 token 消耗，提升 AI 响应质量。

---

## 📐 记忆层级结构

```
┌─────────────────────────────────────────┐
│         MEMORY.md (长期记忆)             │
│   - 用户偏好、重要决策、核心规则          │
│   - 永久保留，每周蒸馏更新               │
│   - 大小：3-10KB 为宜                    │
└─────────────────────────────────────────┘
                    ↑
                    │ 每周蒸馏
                    │
┌─────────────────────────────────────────┐
│    memory/YYYY-MM-DD.md (短期记忆)       │
│   - 当日对话、临时决策、TODO             │
│   - 保留 7 天，超期归档                   │
│   - 每个文件：1-20KB                     │
└─────────────────────────────────────────┘
                    ↑
                    │ 每日追加
                    │
┌─────────────────────────────────────────┐
│   sessions/*.jsonl (原始会话)            │
│   - 完整对话记录                         │
│   - 可选索引，按需检索                   │
│   - 定期清理                             │
└─────────────────────────────────────────┘
```

## 🛠️ 工具与脚本

### 1. 自动蒸馏脚本

**位置**: `workspace/scripts/memory-distill.py`

**功能**:
- 扫描最近 7 天日志
- 提取关键信息（决策、教训、偏好、技能、配置）


### 2. 每周定时任务

**已配置**: 每周日 23:00 自动执行记忆维护

**任务内容**:
1. 归档 7 天前的日志到 `memory/archive/`
2. 扫描本周日志，提取模式更新 MEMORY.md
3. 清理临时文件

**管理命令**:
```bash
# 查看定时任务状态
openclaw cron list

# 手动触发记忆蒸馏任务
openclaw cron run <任务 ID>

# 查看任务执行历史
openclaw cron runs <任务 ID>
```

### 3. HEARTBEAT 检查

**文件**: `workspace/HEARTBEAT.md`

**检查频率**: 每 30 分钟（可配置）

**记忆维护检查项**:
- [ ] 运行记忆蒸馏脚本
- [ ] 审查蒸馏结果，更新 MEMORY.md
- [ ] 确认 archive/ 目录正常归档

## 📋 标准操作流程 (SOP)

### 每周日 23:00 (自动执行)

```
定时任务触发
    ↓
扫描 memory/ 目录
    ↓
移动 7 天前日志 → archive/
    ↓
读取本周日志 (最近 7 天)
    ↓
提取关键信息 (AI 辅助)
    ↓
生成 MEMORY.md 更新建议
    ↓
发送 Feishu 总结报告
```

### 手动蒸馏 (按需)

```bash
# 1. 运行蒸馏脚本
python3 workspace/scripts/memory-distill.py

# 2. 审查输出，识别重要内容

# 3. 更新 MEMORY.md
# 方法 A: 手动编辑
nano workspace/MEMORY.md

# 方法 B: AI 辅助
openclaw run "根据蒸馏报告更新 MEMORY.md，添加新的用户偏好和教训"

# 4. 验证更新
cat workspace/MEMORY.md | head -50

# 5. 提交变更 (如使用 Git)
cd workspace
git add MEMORY.md memory/archive/
git commit -m "distill: weekly memory update $(date +%Y-%m-%d)"
```

## 💡 最佳实践

### ✅ 推荐做法

1. **定期蒸馏**: 每周至少一次，避免积压
2. **精简 MEMORY.md**: 控制在 10KB 以内，只保留精华
3. **分类清晰**: 使用明确的标题和列表
4. **及时归档**: 7 天前的日志及时移动到 archive/
5. **版本管理**: 使用 Git 追踪 MEMORY.md 变更

### ❌ 避免做法

1. **不要**把所有日志都塞进 MEMORY.md
2. **不要**保留过时的信息（定期清理）
3. **不要**忽略蒸馏报告（失去意义）
4. **不要**在 MEMORY.md 中记录临时 TODO

---

## 🔍 故障排查

### 问题 1: 蒸馏脚本找不到日志文件

**原因**: 日志文件命名格式不正确

**解决**: 确保文件名格式为 `YYYY-MM-DD.md`

### 问题 2: MEMORY.md 增长过快

**原因**: 蒸馏不够频繁或提取过多细节

**解决**: 
- 增加蒸馏频率（每周→每 3 天）
- 调整提取模式，只保留最关键信息
- 手动清理过时内容

### 问题 3: 定时任务未执行

**检查**:
```bash
# 查看 cron 状态
openclaw cron status

# 查看任务列表
openclaw cron list

# 手动触发测试
openclaw cron run <任务 ID>
```

## 📈 效果评估

### 优化前 vs 优化后

| 指标 | 优化前 | 优化后 | 改善 |
|------|--------|--------|------|
| MEMORY.md 大小 | 不稳定 | 3-10KB | ✅ 可控 |

## 🚀 进阶扩展

### 1. AI 辅助蒸馏

使用 AI 自动分析日志并生成更新建议：

```bash
openclaw run "读取 memory/2026-03-*.md 文件，提取关键决策和教训，生成 MEMORY.md 更新建议"
```

### 2. 跨会话记忆同步

如未来升级到 Mem0：
- 配置 Mem0 插件
- 迁移现有 MEMORY.md 内容
- 启用跨设备记忆共享

### 3. 记忆质量评分

添加记忆价值评估：
- 高频引用 → 保留
- 30 天未引用 → 标记为待清理
- 过时信息 → 归档或删除
